关于JavaScript的异步编程

发表于 2020-08-10 分类于 JavaScript 阅读次数：

一、同步操作和异步操作

假设你现在正在浏览一个网页，你点击了网页的某一个按钮，向服务端发送某些请求。

  • 如果是同步操作：你需要等待服务端响应，直到它处理完你的请求之后，你才能进行其它操作。在这个等待响应的过程中可能会出现页面“假死”的情况（其实程序正在执行中）。这样导致的结果就是用户体验差。
  • 如果是异步操作：在你发送完请求之后，不需要等待服务端响应。你可以进行其它任意操作，用户体验好。

典型的异步操作有：定时器、ajax…

异步操作那么好，那它有什么缺点呢——编写的代码比较混乱，典型的问题就是异步编程导致的“回调地狱”。

三、Promise对象

1. Promise 是什么

简单来说，Promise 是个容器，让我们对异步操作进行封装。而在语法上，Promise是个对象。

Promise 对象有三种状态：

  • pending 进行中
  • resolved 成功
  • rejected 失败

Promise 对象的状态只有两种变化的情况：

  • pending → resolved
  • pending → rejected

关于 Promise 的一些注意事项：

  1. Promise 容器本身不是异步的， 但是它内部往往都会封装异步任务。
  1. Promise 新建后立即执行，而then方法指定的回调函数在当前脚本所有同步任务执行完才会执行。

3. 通过then方法解决回调地狱

Promise 实例的then方法返回的是一个新的Promise实例（注意，不是原来那个Promise实例）。因此可以采用链式写法，即then方法后面再调用另一个then方法。

第一个then方法的回调函数完成以后，根据返回结果有不同的情况：

  • （常用）若返回结果为 Promise 对象，则第二个 then 方法指定的回调函数，就会等待这个新的 Promise 对象状态发生变化。如果变为 resolved，就调用第一个回调函数，如果变为 rejected，就调用第二个回调函数。
  • （少用）若返回结果不为 Promise 对象，如普通的字符串'hello world'。则该返回结果会作为参数，传入第二个then方法的第一个回调函数。若没有返回值，则返回结果默认为 undefined。

四、async和await

如果说promise是让我们用来对异步操作进行封装

那么async和await就是让我们用同步的代码格式来写异步操作

1. 基本用法

  • aysnc函数的返回值是个 Promise 对象。

  • awiat后面跟的是一个 Promise 对象，返回该 Promise 对象的结果（相当于拿到 resolve 函数传出去的值）

  • await只能出现在async函数中。在函数执行时，一旦遇到await函数会先返回，等到异步操作完成后，再接着执行函数体后面的语句。

  • 若有多个await，程序的执行顺序就和代码的编写顺序一样（汲取了同步的优势，代码简洁）

  • 若await后面的 Promise 对象变为 rejected状态，则整个async函数会中断执行。若不想中断执行，且想获得错误信息，可以通过try catch来捕获异常。
